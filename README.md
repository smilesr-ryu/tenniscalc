# 테니스 계산기 기반 로또 번호 추천기

## 1. 개요

### 프로젝트명
**Tennis Calculator - 가장 많이 사용하는 숫자 기반의 로또 번호 추첨기**

테니스 동호인을 위해 만들어진 통합 계산기 웹앱으로, 사칙 연산 과정에서 사용된 숫자를 추적하고 가장 많이 사용한 숫자를 기반으로 로또 번호를 추천받을 수 있습니다.

### 실행 방법
1. **계산기 사용**: 숫자를 누르고 사칙 연산 수행
2. **가장 많이 사용된 번호 확인**: 화면 아래에서 자동으로 업데이트되는 가장 많이 사용한 숫자 확인
3. **로또 번호 생성**: "🎰 로또 번호 생성" 버튼 클릭
4. **추천 로또 번호 확인**: 생성된 로또 번호(메인 6개 + 보너스 1개) 확인

### 핵심 로직 / API 설명

**데이터 흐름:**
```
사칙 연산 계산 기입력 → 각 숫자 카운팅 → 가장 많이 사용된 숫자 추출
                ↓
        API 전송 (가장 많이 사용된 숫자)
                ↓
        FastAPI 서버가 로또 번호 생성
                ↓
        생성된 로또 번호 응답 및 UI에 표시
```

**특징:**
- 사칙 연산 계산을 하면 입력된 각 숫자가 자동으로 카운팅됨
- 실시간으로 가장 많이 사용된 숫자가 업데이트됨
- 이 숫자를 포함하는 로또 번호를 FastAPI 서버에서 생성해 반환
- 사용자는 본인이 계산 과정에서 가장 선호하는 숫자 기반의 로또 번호를 받음

### 2.3 API

#### FastAPI 로또 생성 서버

**파일:** `api_server.py`

**엔드포인트:**

##### POST `/api/lotto`

**요청 (Request):**
```json
{
  "most_digit": "5",
  "count": 7
}
```

**요청 파라미터:**
| 파라미터 | 타입 | 설명 |
|---------|------|------|
| most_digit | string | 가장 많이 사용한 숫자 (0-9) |
| count | int | 해당 숫자를 사용한 횟수 |

**응답 (Response):**
```json
{
  "main_numbers": [3, 12, 25, 31, 38, 42],
  "bonus_number": 15,
  "most_digit": "5",
  "count": 7
}
```

**응답 필드:**
| 필드 | 타입 | 설명 |
|------|------|------|
| main_numbers | array[int] | 로또 메인 번호 6개 (1-45 범위, 정렬됨) |
| bonus_number | int | 로또 보너스 번호 (메인과 다름) |
| most_digit | string | 요청에서 전달받은 가장 많이 사용한 숫자 |
| count | int | 요청에서 전달받은 사용 횟수 |

**에러 응답:**
```json
{
  "detail": "가장 많이 누른 숫자는 0-9 범위여야 합니다"
}
```

**HTTP 상태 코드:**
| 상태 코드 | 설명 |
|---------|------|
| 200 | 로또 번호 생성 성공 |
| 400 | 잘못된 입력 (숫자 범위 초과 등) |
| 500 | 서버 에러 |

##### GET `/health`

**설명:** API 서버 헬스 체크

**응답:**
```json
{
  "status": "ok"
}
```

## 10. 문제 해결 (Troubleshooting)

### Q1: "API 서버에 연결할 수 없습니다" 에러

**원인:** FastAPI 서버가 실행되지 않음

**해결:**
```bash
# 터미널 1에서 서버 실행 확인
python api_server.py

# 서버가 http://127.0.0.1:8000에서 실행 중인지 확인
# 브라우저에서 http://127.0.0.1:8000/health 접속해보기
```

### Q2: 로또 번호가 생성되지 않음

**원인:** 숫자를 누르지 않은 상태에서 생성 시도

**해결:**
- 계산기에서 숫자를 먼저 누르기 (예: 5 + 3 = )
- 화면 아래에서 "가장 많이 누른 숫자" 확인 후 생성 버튼 클릭

### Q3: 버튼의 `-` 와 `+` 기호가 보이지 않음

**설명:** UI 스타일 이슈 (우선순위 낮음, 향후 개선 예정)

**현재 상태:** 기호는 동작하지만 시각적으로 보이지 않을 수 있음
